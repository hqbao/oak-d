# Gold Sessions — Regression Suite for SkySLAM

**Purpose**: A fixed set of "gold" sessions that every future pipeline version
(BasaltVIO+RTABMapSLAM today, skyslam_backend tomorrow, skyslam_frontend the
day after) must pass. This is the **single source of truth** for regression.

**Principles**:
- Record once, use forever. Do NOT re-record unless calibration changes.
- Folder `sessions/gold/` is gitignored (heavy data) — back up manually to a
  separate drive.
- Baseline numbers are frozen in `GOLD_BASELINE.md` (generated by
  `tools/baseline_report.py` after recording finishes).
- When writing a new module → re-run on all gold sessions and compare against
  the baseline. Regression = numbers worse than baseline → fail.

---

## Scenarios

12 scenarios cover the different "axes" of VIO/SLAM stress. The first 6 are the
original core suite; the second 6 were added later as targeted stress cases for
the from-scratch `ours` pipeline (phantom translation, scale under fast motion,
textureless surfaces).

| ID | Name | Duration | Purpose |
|---|---|---|---|
| 1 | `lab_static_10s` | 10s | bias drift, IMU noise floor — must stand still |
| 2 | `lab_straight_20s` | 20s | translation scale, IMU-visual sync |
| 3 | `lab_loop_30s` | 30s | loop closure (small closed loop, ~3 m radius) |
| 4 | `corridor_60s` | 60s | long-range drift, distant loop closure (~15 m loop) |
| 5 | `quick_motion_15s` | 15s | tracking robustness (fast shake + head turn) |
| 6 | `loop_closure_45s` | 45s | C4 keyframes + loop closure DB persistence |
| 7 | `still_15s` | ~16s | stationary honesty check (no phantom drift when not moving) |
| 8 | `push_straight_fast_15s` | ~16s | translation scale under fast forward push |
| 9 | `push_fwdback_20s` | ~21s | forward-then-back translation, scale symmetry |
| 10 | `push_shake_20s` | ~21s | hard shake — low-parallax phantom-translation stress |
| 11 | `whitewall_move_15s` | ~16s | moving at a textureless wall — PnP inlier-gate stress |
| 12 | `yaw_inplace_15s` | ~17s | in-place yaw — rotation-only, phantom-translation case |

### How to record each scenario

#### 1. `lab_static_10s` — stand still
Put the camera on a tripod or table and DO NOT touch it. Goal: pose must be
~constant; any drift = wrong IMU bias.

```bash
.venv/bin/python tools/record_session.py sessions/gold/lab_static_10s \
    --duration 10 -f
```

Expected: |pos drift| < 5 cm, |rot drift| < 1°.

#### 2. `lab_straight_20s` — walk straight
Hold the camera by hand, walk straight ~5 m, stop, rotate 180°, walk back to
the starting point. Goal: scale + continuous tracking, NO loop closure (not
back close enough to trigger one).

```bash
.venv/bin/python tools/record_session.py sessions/gold/lab_straight_20s \
    --duration 20 -f
```

Expected: ATE VIO-vs-SLAM < 10 cm (no loop, so they should be near equal).

#### 3. `lab_loop_30s` — small closed loop
Walk one loop around a table / desk (~3 m radius) and return exactly to the
starting point. Goal: trigger loop closure → odom_correction must show one
clear jump.

```bash
.venv/bin/python tools/record_session.py sessions/gold/lab_loop_30s \
    --duration 30 -f
```

Expected: ≥1 loop event in `loop_events.jsonl`, ATE SLAM < ATE VIO.

#### 4. `corridor_60s` — long corridor
Walk down a corridor ~10–15 m, turn around, walk back. Goal: accumulated
drift over distance, loop closure at long range.

```bash
.venv/bin/python tools/record_session.py sessions/gold/corridor_60s \
    --duration 60 -f
```

Expected: noticeable VIO drift (~30 cm – 1 m), SLAM corrects to <20 cm after
the loop.

#### 5. `quick_motion_15s` — heavy shake
Hold the camera and walk in a fast circle + continuous head turns + sudden
direction changes. Goal: stress the feature tracker under motion blur + IMU
saturation.

```bash
.venv/bin/python tools/record_session.py sessions/gold/quick_motion_15s \
    --duration 15 -f
```

Expected: no tracking_lost events (if any occur = pipeline is weak, document
the gap count).

#### 6. `loop_closure_45s` — closed loop with loop closure (C4 test)
Stand still 5 s while pointing at a distinct landmark (table corner, poster),
walk a small loop ~2–3 m radius for ~30 s, return to **exactly** the original
position **and** heading, then stand still 5–10 s so RTABMap can lock the
loop. Goal: verify `rtabmap.db` persistence + `extract_kf_from_db.py`
producing ≥1 loop link.

```bash
.venv/bin/python tools/record_session.py sessions/gold/loop_closure_45s \
    --duration 45 -f
```

Expected: ≥1 `kf_loops` (Link type 1/2/3 in rtabmap.db). If = 0 → re-record
(wrong return pose, BoW did not match).

## After recording all 6

1. Back up `sessions/gold/` to an external drive (USB/cloud). It is not
   git-tracked; if lost, it is gone.
2. Generate the baseline report:
   ```bash
   .venv/bin/python tools/baseline_report.py sessions/gold/ \
       > docs/GOLD_BASELINE.md
   ```
3. Commit `docs/GOLD_BASELINE.md` (small file, metric numbers only).

From then on, each time a new skyslam module is written:
```bash
# Re-process gold sessions with skyslam
.venv/bin/python tools/replay_skyslam.py sessions/gold/lab_loop_30s
# Compare against baseline
.venv/bin/python tools/compare_sessions.py \
    sessions/gold/lab_loop_30s/basalt/slam_pose.jsonl \
    sessions/gold/lab_loop_30s/skyslam/slam_pose.jsonl
```

(`replay_skyslam.py` not written yet — will be added in Phase B/C.)
